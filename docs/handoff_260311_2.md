# handoff_260311_2.md

## Agent-Modeler (agmod) — v0.1 MVP (Audited Spec)

---

# 0. Context

This document defines the **implementation contract** for the first working version of **agmod**.

agmod is a **local LLM block manager** that allows users to browse reusable workflow blocks and install them into a project repository.

This document replaces earlier design notes and should be treated as the **authoritative specification for v0.1**.

---

# 1. Goals

The first version of agmod must allow users to:

1. Register directories containing reusable LLM blocks.
2. Discover those blocks recursively.
3. Browse them in a Textual TUI.
4. Copy blocks into the current repository under `/llm/`.
5. Maintain a section in `AGENTS.md` listing those blocks.

---

# 2. Non-Goals

The following features are explicitly out of scope for v0.1:

* Running AI agents
* Workflow orchestration
* Remote block registries
* Block versioning
* Dependency graphs
* Block editing
* Cloud services
* Databases

agmod is **file-based and local only**.

---

# 3. Terminology

## Block

A reusable file discovered in a source directory.

Examples:

```
toolboxes/python_toolbox.md
toolboxes/marimo_toolbox.md
workflows/app-dev/curated_llm_workflow_v1.md
```

Blocks are typically Markdown files.

---

## Source

A registered directory that contains blocks.

Sources are treated as **read-only libraries**.

---

## Project Block

A block copied into the current repository.

Stored under:

```
/llm/
```

Example:

```
llm/toolboxes/python_toolbox.md
```

---

# 4. Repository Detection

agmod must detect the **project root** as:

```
current working directory
```

Assumption:

User runs agmod from the project root.

If `/llm/` does not exist, it must be created automatically.

---

# S-260311-1 — Configuration System

Configuration file location:

```
~/.config/agmod/config.toml
```

Example:

```
[sources]

personal = "/home/user/llm"
workflows = "/home/user/workflows"
```

Rules:

* Keys are source names
* Values are absolute paths
* Directories must exist
* Invalid sources are ignored with warning

If config file does not exist:

Create empty config automatically.

---

# S-260311-2 — Block Discovery

agmod must recursively scan source directories.

Discovery rules:

* recursive scan
* include `.md` files
* ignore hidden directories
* store relative path from source root

Example block record:

```
source = personal
relative_path = toolboxes/python_toolbox.md
absolute_path = /home/user/llm/toolboxes/python_toolbox.md
```

---

# S-260311-3 — Metadata Extraction

Metadata fields:

```
name
description
tags
```

Extraction priority:

1. YAML frontmatter
2. first heading
3. filename

Fallback must never crash.

---

# S-260311-4 — Project Block Storage

Blocks copied into:

```
/llm/
```

Rules:

* preserve directory structure
* create directories automatically
* allow overwrite with confirmation

Example:

```
source: toolboxes/python_toolbox.md
target: llm/toolboxes/python_toolbox.md
```

---

# S-260311-5 — AGENTS.md Integration

agmod must maintain a managed section.

Section format:

```
### LLM Context Blocks

These files contain reusable instructions and workflows
that may be relevant when working in this repository.

<!-- agmod:start -->

- llm/toolboxes/python_toolbox.md
- llm/toolboxes/marimo_toolbox.md

<!-- agmod:end -->
```

Rules:

* only edit between markers
* rewrite region completely
* preserve all other content

---

## Marker Behavior

If markers exist:

Update contents between them.

If markers do not exist:

Insert section **near top of file**, after initial header lines.

Example insertion point:

```
# AGENTS.md
# Guidance for agentic coders working in this repo.

[INSERT BLOCK HERE]

## 0) Sources of truth inside this repo
```

If `AGENTS.md` does not exist:

Skip integration silently.

# S-260311-6 — TUI Interface

Application launches with:

```
uv run agmod
```

Framework:

```
Textual
```

Layout:

```
┌──────────────┬──────────────┬──────────────┐
│ Sources      │ Project      │ Info         │
│              │ Blocks       │ Panel        │
└──────────────┴──────────────┴──────────────┘
```

---

## Sources Panel

Displays block tree grouped by source.

Example:

```
personal/
  toolboxes/
    python_toolbox.md
    marimo_toolbox.md
```

---

## Project Panel

Displays blocks present in:

```
/llm/
```

---

## Info Panel

Displays:

```
name
source
path
description
preview
```

Preview may display first 10–20 lines.

# S-260311-7 — Operations

Supported operations:

Add block

```
copy source → project
update AGENTS.md
```

Remove block

```
delete project file
update AGENTS.md
```

Refresh

```
rescan sources
```

agmod v0.1 is complete when:

* `uv run agmod` launches the Textual interface
